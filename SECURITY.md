# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please email: **[security@shelvance.org](mailto:security@shelvance.org)**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes

## Response

We will respond and work on a fix as soon as possible. Security issues are taken seriously and handled with best effort.

## Supported Versions

Only the latest release receives security updates. Always run the latest version.

## Disclosure

- Issues are investigated privately
- Fixes are released before public disclosure
- Credit given to reporters (unless anonymity requested)

Thank you for helping keep Shelvance secure.
